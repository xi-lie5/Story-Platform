## 1. 项目概览

StoryForge 是一个 AI 互动故事创作平台，采用前后端一体化架构：

```
Story-Platform-main/
├── backend/                  # Express.js 后端
│   ├── server.js             # 入口，注册路由 + 启动服务
│   ├── config/database.js    # MySQL 连接池 + 表创建
│   ├── models/               # Active Record 模型
│   ├── routes/               # 路由处理
│   ├── middleware/            # auth、adminAuth、errorHandler
│   ├── utils/                # 工具函数
│   └── scripts/              # 管理脚本
├── front/                    # 纯静态前端
│   ├── *.html                # 14 个页面
│   └── assets/js/            # 共享 JS 模块
│       ├── api-config.js     # API 配置 + 请求工具
│       ├── auth.js           # AuthUI 认证组件
│       └── navbar.js         # 导航栏动态渲染
├── docs/                     # 项目文档
├── docker-compose.yml        # MySQL 容器
└── .gitignore
```

**技术栈（全部来自 `package.json` 和实际代码）：**

| 层 | 技术 | 版本 |
|----|------|------|
| 运行时 | Node.js | v14.0.0+ |
| Web 框架 | Express.js | v5.1.0 |
| 数据库 | MySQL | 8.0+ (Docker) |
| 数据库驱动 | mysql2 | v3.11.0 |
| 认证 | jsonwebtoken | v9.0.2 |
| 密码加密 | bcryptjs | v3.0.2 |

| AI 服务 | DeepSeek Chat API (deepseekClient.js) + 阿里云 DashScope (qwen-max) |
| 前端 CSS | Tailwind CSS (CDN) | v3 |
| 图标 | Font Awesome | 6.0.0 (AI 页面) / 4.7.0 (my_stories) |

## 2. 环境搭建

### 2.1 前置要求

- Node.js v14+ (`node --version`)
- Docker Desktop（用于 MySQL）
- Git

### 2.2 初始化步骤

```bash
# 1. 克隆仓库
git clone https://github.com/xi-lie5/Story-Platform.git
cd Story-Platform

# 2. 启动 MySQL
docker compose up -d

# 3. 确认 MySQL 运行
docker ps --filter "name=mysql"

# 4. 安装依赖
cd backend && npm install

# 5. 配置环境变量（已有 .env 文件，确认以下字段）
# MYSQL_HOST=localhost
# MYSQL_PORT=3306
# MYSQL_USER=root
# MYSQL_PASSWORD=your_password
# MYSQL_DATABASE=story_platform
# JWT_SECRET=your_jwt_secret_min_32_chars
# DEEPSEEK_API_KEY=your_deepseek_key
# DASHSCOPE_API_KEY=your_dashscope_key

# 6. 启动服务（自动创建表结构）
node server.js
# 输出: 🚀 服务运行在 http://localhost:5000/api/v1

# 7. 创建管理员账号
node scripts/createAdmin.js
```

### 2.3 前端访问

前端是纯静态 HTML，通过 Express 静态文件中间件提供服务：

```
http://localhost:5000/                 → 首页 (index.html)
http://localhost:5000/ai-workshop.html → AI 工作坊
http://localhost:5000/ai-stories.html  → AI 故事广场
http://localhost:5000/login.html       → 登录
```

## 3. 代码规范

### 3.1 JavaScript 风格（基于项目现有代码）

**变量声明：** 使用 `var`（项目现有风格，保持一致性）
```javascript
// ✅ 项目风格
var storyId = null;
var characters = [];

// ❌ 不要混用 let/const（除非在新建模块中）
```

**函数声明：** 使用 `function` 关键字
```javascript
// ✅ 项目风格
async function createAIStory() { ... }
function escapeHtml(text) { ... }
```

**字符串：** 使用单引号（项目现有风格）
```javascript
// ✅
var url = API_CONFIG.buildUrl('/aiStory/configs');
// ❌
var url = API_CONFIG.buildUrl("/aiStory/configs");
```

### 3.2 后端约定

**路由文件结构：** 每个文件导出一个 Express Router
```javascript
const express = require('express');
const router = express.Router();
router.get('/path', authGuard, [...validators], async (req, res, next) => { ... });
module.exports = router;
```

**错误处理：** 统一使用 `errorFormat`
```javascript
// ✅ 正确的错误返回
return next(errorFormat(400, '描述', [{ field, message }], 错误码));
return next(errorFormat(404, '资源不存在', [], 10002));
return next(errorFormat(500, '服务器错误: ' + error.message, [], 10013));
```

**数据库查询：** 
- 🔴 含 `LIMIT`/`OFFSET` 的查询：必须用 `conn.query()`，LIMIT 值直接拼入 SQL
- 🟢 不含 LIMIT/OFFSET 的查询：可用 `conn.execute()`
```javascript
// ✅ 含 LIMIT: 用 query() + 整数拼接
const [rows] = await conn.query(
  'SELECT * FROM ai_stories WHERE status = ? LIMIT ' + limit + ' OFFSET ' + offset,
  ['published']
);

// ✅ 不含 LIMIT: 可用 execute()
const [rows] = await conn.execute('SELECT * FROM users WHERE id = ?', [userId]);
```

**事务：** 所有写操作（涉及多表）必须使用事务
```javascript
const conn = await pool.getConnection();
try {
  await conn.beginTransaction();
  // ... writes ...
  await conn.commit();
} catch(e) {
  await conn.rollback();
  throw e;
} finally {
  conn.release();
}
```

### 3.3 前端约定

**导航栏：** 所有页面使用统一结构（见 `navbar.js`）
- 桌面端：`#main-nav-desktop` 容器
- 移动端：`#mobile-menu` 容器
- 认证区域：`#auth-area-desktop` + `#auth-area-mobile`

**API 调用：** 统一使用 `API_CONFIG` 对象
```javascript
// ✅
var res = await API_CONFIG.get(API_CONFIG.buildUrl('/aiStory/configs'));
var res = await API_CONFIG.post(API_CONFIG.buildUrl('/aiStory/config'), payload);
var res = await API_CONFIG.patch(API_CONFIG.buildUrl('/aiStory/configs/' + id + '/publish'), body);
```

**HTML 转义：** 所有页面引用 `assets/js/utils.js`（待创建）中的 `escapeHtml()`

## 4. Git 工作流

### 4.1 分支策略

```
main ← feature-xxx ← dev-work
      ↑              ↑
      永远可部署      日常开发在这里
```

**规则：**
- `main` 分支永远可部署，不允许直接在 main 上开发
- 每个功能/修复一个 feature 分支，命名格式：`feature/<模块名>-<简述>`
- 示例：`feature/comments-api`、`feature/export-pdf`、`fix/execute-query-bug`

### 4.2 Commit 规范

```
<type>: <简短描述>

<详细说明（可选）>

Co-Authored-By: Claude <noreply@anthropic.com>
```

**type 取值：**
- `feat`: 新功能
- `fix`: 修复 bug
- `refactor`: 重构（不改变行为）
- `chore`: 杂项（CI、文档、构建）
- `test`: 测试

**示例：**
```
feat: add story comment system with per-node commenting

- POST /stories/:id/comments endpoint
- GET /stories/:id/comments with pagination
- Comment list UI in story reader

Co-Authored-By: Claude <noreply@anthropic.com>
```

### 4.3 每日流程

```bash
# 早上
git checkout main && git pull
git checkout -b feature/your-module

# 开发中
git add -A && git commit -m "feat: ..."

# 下班前
git push origin feature/your-module
# 在 GitHub 开 Draft PR
```

## 5. 关键踩坑记录

> 以下全是在本项目中实际遇到的 bug。

| 问题 | 原因 | 解决 |
|------|------|------|
| `mysqld_stmt_execute` 错误 | `conn.execute()` 不支持参数化 `LIMIT ? OFFSET ?` | 改用 `conn.query()` + 整数拼接 |
| `ai_stories` 表缺列 | `CREATE TABLE IF NOT EXISTS` 不修改已有表 | 引入迁移系统（Phase 1 模块 2） |
| `escapeHtml` 定义 5 次 | 每个页面独立复制 | 抽取到 `utils.js`（Phase 1 模块 1） |
| `my_stories.html` 导航栏不同 | 该页面使用独立的浅色主题 | 统一使用深色主题 + navbar.js（Phase 1） |
| `loadConfigForEdit` 用 `setTimeout` 等分类加载 | 异步时序问题 | 改用 Promise 回调链（Phase 2 模块 5） |
