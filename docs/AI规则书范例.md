# AI 规则书（故事配置）范例

本文档提供一份**可直接照抄使用**的 AI 规则书范例，以及每个字段的填写说明和技巧，帮助你在「AI 故事工坊」（`ai-workshop.html`）中创建出效果更好的 AI 互动故事。

> AI 规则书 = 你在工坊 5 个步骤中填写的所有配置，最终会被拼装成一段 System Prompt 喂给 DeepSeek，AI 严格依据这份"设定书"生成剧情和选项。**设定越具体、越有画面感，AI 生成的内容就越稳定、越贴合你的预期。**

---

## 一、字段说明与技巧

| 工坊步骤 | 字段 | 是否必填 | 说明 |
|---|---|---|---|
| 1 | 故事标题 | ✅ | 简短有吸引力，会展示在广场列表 |
| 1 | 故事分类 | 可选 | 用于广场筛选 |
| 1 | 开端提示（startPrompt） | ✅ | **AI 生成第一章的直接指令**，建议写清楚"开场画面+主角状态+一个钩子（悬念/冲突）" |
| 2 | 世界观设定（worldSetting） | ✅ | 时代背景、世界规则、社会结构。**这是最重要的字段**，AI 之后每一步续写都会参考它 |
| 2 | 氛围标签 | 可选 | 黑暗/轻松/史诗/温馨/紧张，影响整体基调 |
| 3 | 角色卡（characters） | ✅ 至少1个 | 每个角色：姓名、性格标签、描述、（可选）关系网。主角也建议加一张卡 |
| 4 | 故事大纲（outline） | 可选但强烈建议填 | 列出关键情节点（3-6条），能显著降低"AI 跑题/虎头蛇尾"的概率 |
| 4 | 叙事视角 | 有默认值 | 第一人称沉浸感强，第三人称更好控制多角色 |
| 4 | 文风基调 | 有默认值 | 自然流畅/诗意悠远/紧张刺激/幽默诙谐/冷峻克制/温暖治愈 |
| 4 | 篇幅控制 | 有默认值 | 标准 800-1500字/段，建议新手用"标准"，太长容易导致 AI 输出被截断 |

**技巧总结：**
1. **世界观要具体到"规则"层面**，不要只写"这是一个魔法世界"，要写清楚魔法的代价、社会如何看待魔法、有哪些势力。
2. **开端提示 = 一个具体场景 + 一个未解决的问题**，避免空泛的"某一天，主角开始了冒险"。
3. **角色关系网（relationships）** 能让 AI 更好地处理多角色互动（如"暗恋""上下级""宿敌"）。
4. **大纲不是死板剧本**，AI 会围绕这些节点灵活展开，但不会完全偏题。

## 二、完整范例：《雾隅事务所》（悬疑/都市异能）

以下范例可直接在工坊中逐步骤填入。

### 步骤 1：基本信息

- **故事标题**：`雾隅事务所`
- **故事分类**：悬疑 / 都市（如分类不存在可留空）
- **开端提示（startPrompt）**：
```
深秋的雨夜，林晚作为"雾隅事务所"新入职的调查员，第一次独自出勤。
她被派去调查一栋老公寓——三天前，613室的租客在门锁完好、监控无异常的情况下离奇失踪，
只在桌上留下一张手写的字条："雾进来了。"
林晚站在613室门口，钥匙插进锁孔的瞬间，走廊的灯忽然全部熄灭。
```

### 步骤 2：世界观设定

- **世界观设定（worldSetting）**：
```
故事发生在现代都市"临江市"，表面与现实无异，但城市中散落着若干"雾隅"——
一种因强烈负面情绪（怨恨、恐惧、悔恨）聚集而形成的空间异常，外人称之为"雾"。
雾隅内的时间、物理规则会被扭曲，进入者若不能在雾消散前找到"心结"并化解，
就会永远留在雾中，从现实世界中"消失"，如同从未存在过。

雾隅事务所是临江市唯一公开注册、专门处理雾隅事件的民间机构，游走在警方和市政厅的灰色地带。
事务所调查员必须具备"感雾体质"（能察觉雾隅边界的人），这类人在人群中约万分之一。
每一次进入雾隅都伴随记忆侵蚀的风险——调查员可能会带着雾隅中"别人的记忆"走出来。

社会对雾隅事件普遍保持沉默和回避态度，警方会将失踪案草草结案，
只有事务所和少数受害者家属知道真相。
```

- **氛围标签**：`黑暗`、`紧张`

### 步骤 3：角色工坊

| 角色名 | 性格标签 | 描述 |
|---|---|---|
| 林晚 | 冷静, 敏感, 有心理阴影 | 事务所新人调查员，感雾体质者。三年前妹妹在一次雾隅事件中失踪，这是她加入事务所的真正原因。表面理性克制，内心一直在寻找妹妹的痕迹。 |
| 沈屿 | 神秘, 寡言, 经验丰富 | 事务所的资深调查员，林晚的搭档兼指导人。已经处理过上百起雾隅事件，身上有无法解释的"记忆错位"痕迹，似乎隐瞒着与雾隅有关的秘密。 |
| 老周 | 圆滑, 讲实际, 略市侩 | 事务所所长，看似只关心委托费用，实际上一直在暗中庇护调查员、周旋于警方与雾隅受害者家属之间。 |

角色关系网（在角色卡的"关系"字段填写，示例）：
```
林晚 → 沈屿：既依赖又警惕（怀疑他知道妹妹失踪的真相）
沈屿 → 林晚：愧疚与保护欲（似乎认识林晚失踪的妹妹）
老周 → 两人：表面上是雇主，实际上更像守护者
```

### 步骤 4：大纲与风格

- **故事大纲（outline）**：
```
- 林晚首次独自进入613室的雾隅，发现雾隅内的空间是失踪租客记忆中扭曲的"家"
- 林晚在雾隅中遇到"另一个自己"的幻象，第一次意识到雾隅会利用调查员自身的心结
- 沈屿及时赶到营救，但暴露出他此前曾进入过与林晚妹妹相关的雾隅
- 随着调查深入，事务所发现613室事件与三年前林晚妹妹的失踪案存在同一诱因
- 结局走向：林晚必须选择——揭穿沈屿隐藏的秘密，还是与他联手追查雾隅背后更大的秘密
```

- **叙事视角**：`第三人称`
- **文风基调**：`冷峻克制`
- **篇幅控制**：`标准（800-1500字/段）`

### 步骤 5：确认创建

保存后即可在广场发布，或直接在"我的作品"中点击"开始冒险"体验。

## 四、常见问题

**Q: 世界观写多长合适？**
A: 200-500字为宜。太短 AI 会自由发挥导致跑偏；太长可能挤占续写时的上下文空间。

**Q: 角色一定要写"关系"吗？**
A: 强烈建议至少写主角和1个重要配角的关系，能显著提升多角色互动剧情的合理性。

**Q: 大纲会限制 AI 的自由发挥吗？**
A: 不会完全限制。大纲更像"路标"，AI 会围绕这些关键节点展开，但每一步的具体走向仍由你的选择决定。

**Q: 为什么第一章生成后，感觉和我写的开端提示不太一样？**
A: 检查开端提示是否足够具体（场景+人物+钩子）。越具体，AI 的发挥空间被约束得越好，生成结果越贴近预期。
